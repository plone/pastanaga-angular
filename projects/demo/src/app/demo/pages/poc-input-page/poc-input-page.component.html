POC Form field Inputs

This POC is a form field with the same behavior as Pastanaga's Input, but with full support of:

  • Angular ngModel
  • Angular reactive forms

this page helps testing it.

Detailed feature description

List of pastanaga's feature that should be maintained:

BaseControl

  • id is an optional input and is autogenerated when not provided.

  • name is an optional input and is autogenerated when not provided.

  • help is an input providing tips and displayed in the pa-field container.

  • disabled forces internal state.
    NB: should not be used with ngModel or reactiveForms as disabled state is handled by angular (a warning message is provided by angular)

BaseTextField

  • value ineffective when used in combination with ngModel or reactiveForm with a warning message

  • debounceDuration is used to configure a debounced change emission.

  • placeholder input's placeholder

  • readonly html property.

  • errorMessages list of messages corresponding to standard validations.
    NB: if a validator provides a message, validator's message will be displayed, not the one in this object. Ex: 

    
                    parentControl = new FormControl('', [myCustomValidator]);
                    ...
                    function myValidator(formControl)  {
                        if (!formControl.value)
                            return { required: 'you absolutely need to fill this one !'};
                        return null;
                    }

    should display: you absolutely need to fill this one !

  • pattern regex internal validation.

  • errorMessage forces internal error state with the given error message

  • required internal validation.

  • valueChange a debounced value emission, will be renamed as debouncedValueChange

  • instantValueChange value change emission, will be renamed as valueChange

  • keyUp provides the current value on keyUp

  • enter provides the current value when enter key is keyup


  • blurring

    blur event.

  • focusing

InputComponent
  • type
  • acceptHtmlTags
  • noAutoComplete
  • hasFocus
  • min
  • max
  • maxlength
New expected behavior
  • Internal validation and parent's validators should be merged to validte input.
  • Dynamically changing internal or parent validation should be supported.
  • Standard behavior: validation is not displayed when field is pristine.
