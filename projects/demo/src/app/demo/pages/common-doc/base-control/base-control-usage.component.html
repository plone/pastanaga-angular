Inputs

describedBy: string
Reference to an element describing the field (optional).
disabled: boolean
Set disabled state (default: false)
errorMessage: string
Error message to display when field validation is done externally. Will be added to internal validation error if any. (optional)
errorMessages: IErrorMessages
All possible basic error messages (like required) (optional: by default field will have the error state without any message displayed)
hasFocus: boolean
Set to true to focus on the html native element (default: false)
help: string
Field's help (optional). Only displayed when the form field is VALID. Used as aria-describedby reference when none is provided.
id: string
Form field's id (optional). The html form element will have an id based on the provided value. If the value is not provided, an auto-generated id will be applied to the html nativeElement.
name: string
Form field's name (optional). When provided, the form field carry the value used in the form and the html nativeElement will have no 'name' property. Otherwise, id is used for the 'name' property of the html nativeElement.
readonly: boolean
Sets readonly state (default: false)
showAllErrors: boolean
When true, display all error messages. Display only the the first one (from alphabetical order) otherwise (default: true)
updateOn: UpdateOnStrategy
Reports the update strategy of the AbstractControl (meaning the event on which the control updates itself).(default: 'change'). This can only be changed when form field is in standalone mode, otherwise, parent's update strategy will be applied.
updateValidator: Subject
An event to notify the component when parent's validators have changed. (optional)
value
Field's value. Should only be used when form field is in standalone mode.
