Inputs

describedBy
Reference to an element describing the field, used in aria-describedby (optional) .
disabled
Set disabled state (default: false)
errorMessage
Error message to display when field validation detects an error. (optional)
errorMessages
IErrorMessages defining all possible basic error messages (like required ) (optional: by default field will have the error state without any message displayed)
hasFocus
Set to true to focus on the html native element (default: false)
help
Field's help (optional) . Only displayed when the form field is VALID. Used as aria-describedby reference when none is provided.
id
Form field's id (optional) . The html form element will have an id based on the provided value. If the value is not provided, an auto-generated id will be applied to the html nativeElement.
name
Input's name attribute (optional) . Is required by angular when using template driven forms.
readonly
Sets readonly state (default: false)
showAllErrors
When true, display all error messages. Display only the the first one (from alphabetical order) otherwise (default: true)
updateOn
Reports the AbstractControl 's update strategy (ie the event on which the control updates itself). (type: UpdateOnStrategy, default: 'change') . This can only be changed when form field is in standalone mode, otherwise parent's update strategy will be applied.
updateValidator
An event ( Subject ) to notify the component when parent's validators have changed. (optional)
value
Field's value. Should only be used when form field is in standalone mode.
