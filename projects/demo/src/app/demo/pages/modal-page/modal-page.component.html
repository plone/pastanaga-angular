Modal

Pastanaga provides a full system allowing you to create and manage your modals as you wish.

It also provide two predefined kind of modal component ready to be used:

  • DialogComponent is a blocking modal without closing button containing:
    • an optional image
    • a title
    • a description
    • a footer
  • ModalComponent is a modal with a closing containing:
    • a title
    • a description
    • a content
    • a footer

Pastanaga is providing two ready to use modal components: DialogComponent and ModalComponent.

You can also create your own modal components.

Using DialogComponent or ModalComponent

First you need to create a component implementing IModal interface. IModal interface requires you to have a ViewChild referencing the modal component

{{dialogComponent}}

In its template, you need to use a pa-dialog or a pa-modal

{{dialogTemplate}}

This component being setup, you can then display it using Pastanaga's ModalService

{{callerComponent}}

Creating your own modal component

To create a modal from scratch, you first need to create a component extending BaseModalComponent.

{{customModalComponent}}

In its template, you must identify the modal container with #modalContainer. You can also use pastanaga's classes to take advantage of pastanaga's style and animations. And last but not least, you can use pastanaga's modal directives to setup content location for your modal. 

{{customModalTemplate}}

From there, you can use your own modal the same way you would do with Pastanaga's DialogComponent and ModalComponent
