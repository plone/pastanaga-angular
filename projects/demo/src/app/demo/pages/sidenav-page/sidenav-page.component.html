Sidenav

pa-side-nav component allows to add a side navigation to your app.

The side navigation is always visible on desktop, while it is hidden and displayed only on user action such as clicking on a menu button on mobile and tablet.

 The side navigation of Pastanaga's documentation is build using a sidenav component.

Inputs

mode
Viewport mode: mobile | tablet | desktop
visible
Manage the visibility of the component on mobile and tablet. Ignored in desktop.

Outputs

close
Event emitted when triggering the closing of the side nav on mobile and tablet.

Directives

Header

By default, the sidenav displays only a close button on mobile/tablet.

You can provide your own header by using pa-side-nav-header directive. When doing so, make sure to include a close button if you want one (it's always possible to close the sidenav by clicking on the overlay though)

Content

All the content of a sidenav must be wrapped into pa-side-nav-content directive.

When the content is a list of navigation items, you can use pa-side-nav-item wrapped into an `ul`.

SideNavItem inputs
active
If true, active styles will be applied.
header
If true, the label will be displayed within a h2 tag.
label
Label displayed
