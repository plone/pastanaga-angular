Form control

paFormControl is a directive designed to build custom form elements. It creates or exposes a FormControl:

  • provides consistent id and name.
  • allows consistent form control state management (readonly / disabled / status) on any use case.

Pastanaga’s form fields components (like pa-input, pa-select, pa-checkbox,…) are extending this directive. So all our form field components have a consistent behavior.

Because it is a directive and not a component, it can be used on standard html elements with these three exceptions:

  • standalone fields won't be added to form's values
  • input elements inner change won't be bound to ngModel
  • formControl directives cannot be used on html elements which are not managed by angular reactive form api (for example divs).

Important note: formControl will only be available after the OnInit lifecycle.

When used in combination with ngModel, it exposes the corresponding formControl, allowing to manage its state and validation:
