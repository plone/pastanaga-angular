pa-form-control

Creates or exposes a FormControl

This directive is designed to build custom form elements.

In a standalone usage, it creates a formControl:

{{standaloneCode}}

It exposes the formControl of a given ngModel, allowing to manage it's state and validation:

{{ngModelCode}}

It reflects a given formControl:

{{formControlCode}}

pa-form-control:

  • provide consistent id and name.
  • allow consistent form control state management (readonly / disabled / status) on any use case.

Because it is a directive and not a component, it can be used on standard html elements with these three exceptions:

  • standalone fields won't be added to form's values
  • input elements inner change won't be bound to ngModel
  • formControl directives cannot be used on html elements which are not managed by angular reactive form api (for example divs).

Important note: the formControl will only be available after the OnInit lifecycle.
